## Context

### Original Request
Update Hippocrates from v1.5 to v1.6 with new engine features and system improvements.

### Current State (v1.5)
- ✅ AnalyzerPlugin system for custom detection
- ✅ HippocratesHooks for security event monitoring
- ✅ Modular engine + system architecture
- ✅ 6 Defense Layers (L0-L6) with Redis-backed scoring
- ✅ Honeypot/Decoy system with 4 templates
- ❌ **No Redis graceful degradation** — if Redis fails, app crashes
- ❌ **No IP allowlist** — every IP gets full security check
- ❌ **No stats/observability** — zero metrics exposed
- ❌ **Outdated UA patterns** — missing 2026 AI agent user agents
- ❌ **No body size limits** — huge payloads processed fully
- ❌ **No method-based thresholds** — GET treated same as POST
- ❌ **No config presets** — must configure everything manually
- ❌ **No custom violation messages** — all violations get same decoy

### Research Findings
**New AI Agent UA Patterns (2026):**
- `ClaudeBot/1.0` — Anthropic's training crawler
- `Claude-User/1.0` — User-initiated fetches
- `Claude-SearchBot/1.0` — Search indexing
- `claude-code/` — Claude Code CLI fetches
- `Cursor/` — Cursor IDE agent requests
- `PerplexityBot/` — Perplexity AI search crawler
- `GitHubCopilot/` — Copilot agent requests

**Redis Graceful Degradation Best Practices:**
- Wrap all Redis ops in try-catch
- On failure: log warning, return pass-through (allow request)
- Optionally: circuit breaker pattern (stop retrying after N failures)
- Never let Redis crash cascade to the application
